# Etapa 2 — Onboarding: produto, personas e URLs críticas

## 2.1 O que é o Oftware (em uma frase)

Plataforma de **gestão e acompanhamento** de tratamento (médico, paciente, nutrição e treino), com dashboards, mensagens, pagamentos e relatórios.

## 2.2 Personas que “enxergam” cores diferentes hoje

| Persona | URL principal | Expectativa emocional (marca) |
|---------|---------------|------------------------------|
| **Visitante / lead** | `/` | Confiança, modernidade, clareza |
| **Paciente** | `/meta` | Acolhimento, progresso, calma |
| **Médico** | `/metaadmin` | Profissionalismo, eficiência, dados |
| **Nutricionista** | `/metanutri` | Saúde, organização (hoje muito **verde/esmeralda**) |
| **Personal** | `/metapersonal` | Energia, disciplina (hoje **emerald/teal**) |
| **Operações / super-admin** | `/metaadmingeral` | Controle, neutralidade |

## 2.4 Onde a marca “quebra” hoje (preview do diagnóstico)

1. **Home** usa roxo + laranja + verde + amarelo (por persona).  
2. **Áreas logadas “admin/medico”** usam **verde** como primária.  
3. **MetaNutri / MetaPersonal** usam **esmeralda/teal**.  
4. **Layout Interativo** do paciente usa **laranja/rosa/roxo** nos heróis.

Isso não é “errado”, mas exige **decisão**: uma marca **única** com **acentos por persona**, ou **submarcas** explícitas.

4. Home continua **multicolor** por card ou unifica?
